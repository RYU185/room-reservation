## 컴포넌트 설명

### React SPA (프론트엔드)

브라우저에서 실행되며 API 서버와 REST로 통신합니다.

- **일반 사용자 화면** — 룸 목록 조회, 예약 생성/수정/취소, 캘린더 뷰
- **관리자 대시보드** — 룸 관리, 사용자 관리, 예약 전체 관리, 통계

### API 서버 (Spring Boot)

단일 서버가 모든 비즈니스 로직을 담당하는 모놀리식 구조입니다.

| 모듈 | 역할 |
|------|------|
| 인증 모듈 | 이메일/비밀번호 로그인, Google OAuth2 소셜 로그인, JWT 발급 및 검증 |
| 룸 모듈 | 회의실 목록 조회, 상세 정보, 가용성 확인 |
| 예약 모듈 | 예약 생성/수정/취소, 캘린더 데이터 제공 |
| 관리자 모듈 | 룸·사용자·예약 전체 관리, 통계 조회 |

### PostgreSQL (주 데이터베이스)

모든 영구 데이터를 저장합니다.

| 테이블 | 내용 |
|--------|------|
| users | 사용자 정보 (로컬/OAuth2 통합) |
| rooms | 회의실 정보 |
| reservations | 예약 내역 (시간 중복 방지 exclusion constraint 적용) |
| refresh_tokens | JWT Refresh Token 저장 |

### Redis (캐시)

자주 조회되는 데이터를 캐싱해 DB 부하를 줄입니다.

- 캐시 대상: 룸 목록, 캘린더 데이터, 룸 통계
- 쓰기 발생 시 관련 캐시를 무효화하는 Cache-Aside 패턴 사용

## 인증 흐름

```
이메일 로그인 ──┐
               ├──→ JWT 발급 (Access Token 1h / Refresh Token 7d)
Google OAuth2 ──┘
```

모든 API 요청에 Access Token을 첨부하고, 만료 시 Refresh Token으로 갱신합니다.

## 데이터 흐름

```
읽기: SPA → API 서버 → Redis (캐시 히트)
                            ↓ 미스
                       PostgreSQL → Redis 저장

쓰기: SPA → API 서버 → PostgreSQL 반영 → Redis 캐시 무효화
```

## 접근 제어

| 사용자 유형 | 접근 가능 범위 |
|-------------|---------------|
| 비인증 | 로그인, 회원가입 |
| 일반 사용자 | 룸 조회, 본인 예약 CRUD |
| 관리자 | 전체 룸·사용자·예약 관리 + 통계 |
